Transaction Processing Service

A cloud-native, event-driven microservice for financial transaction processing with idempotency guarantees and distributed state management.

Go 1.22 Terraform AWS Docker

Key Features

🔒

Idempotent Processing

Duplicate requests are safely handled via idempotency keys stored in DynamoDB with automatic TTL-based expiration.

⚡

Optimistic Locking

Version-based concurrency control prevents race conditions when multiple consumers process the same transaction.

🔄

State Machine

Two-phase settlement flow (PENDING → PROCESSING → SETTLED/FAILED) with validated transitions and automatic rollback.

📡

Event-Driven

SQS for async processing, SNS for fan-out notifications, EventBridge for domain event routing across services.

📊

Observability

JSON-structured logging, request correlation IDs, health/readiness endpoints, and CloudWatch integration.

🏗️

Infrastructure as Code

Modular Terraform provisioning for all AWS resources — DynamoDB, SQS, SNS, and EventBridge.

PATCH /api/v1/transactions/:id/settle Settle a pending transaction
GET /health Health check
GET /ready Readiness probe (checks DynamoDB)

Transaction State Machine

Every transaction follows a strict state machine with validated transitions:

                ┌──────────┐
                │ PENDING  │
                └────┬─────┘
                     │
              ┌──────┼──────────┐
              ▼      ▼          ▼
        ┌──────────┐ ┌────────┐ ┌───────────┐
        │PROCESSING│ │ FAILED │ │ CANCELLED │
        └────┬─────┘ └────────┘ └───────────┘
             │
        ┌────┼────┐
        ▼         ▼
  ┌──────────┐ ┌────────┐
  │ SETTLED  │ │ FAILED │
  └──────────┘ └────────┘

Optimistic locking (version field) ensures that concurrent settlement attempts don't corrupt state — only the first write succeeds, others receive a conflict error and can retry.

Quick Start

# Clone and run locally
git clone https://github.com/rahuljain/txn-processing-service.git
cd txn-processing-service

# Start with LocalStack (mock AWS)
docker-compose up -d

# Or run directly
make run

# Run tests
make test

# Deploy infrastructure
cd terraform && terraform init && terraform apply
